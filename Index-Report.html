Web Application Security Vulnerability Report

Target Website: https://rphs.nclack.k12.or.us

Scope: Absence of Anti-CSRF Token and PII Disclosure

1. Absence of Anti-CSRF Token

Overview

Cross-Site Request Forgery (CSRF) is a vulnerability that allows an attacker to trick an authenticated user’s browser into submitting unintended requests. The application does not appear to implement anti-CSRF protections on at least one state-accessible endpoint.

Evidence

Security testing identified the absence of anti-CSRF tokens on the following page:

GET https://rphs.nclack.k12.or.us/about-us/staff-calendar

Requests associated with this page do not include a unique, server-validated anti-CSRF token, indicating that request origin and user intent are not verified.

CWE ID: 352 (Cross-Site Request Forgery)

Impact

If a user is authenticated to the site, an attacker could potentially cause unauthorized requests to be executed by luring the user to a malicious webpage. This may result in unintended actions being performed under the victim’s session.

Remediation

  • Implement unique, unpredictable anti-CSRF tokens for all state-changing requests.
  • Validate tokens server-side before processing requests.
  • Consider validating Origin or Referer headers as an additional safeguard.

2. Personally Identifiable Information (PII) Disclosure

Overview

Personally Identifiable Information (PII) Disclosure occurs when sensitive information is exposed through publicly accessible application responses. Passive analysis identified sensitive numeric data returned from multiple pages.

Evidence

PII was detected in responses from the following endpoints:

GET https://rphs.nclack.k12.or.us/fs/pages/athletics GET https://rphs.nclack.k12.or.us/fs/pages/calendar GET https://rphs.nclack.k12.or.us/fs/pages/news

The exposed data matched patterns consistent with credit-card-related identifiers:

Evidence: 4427313810781538 CWE ID: 359 WASC ID: 13 Source: Passive (PII Disclosure)

Impact

Disclosure of PII on publicly accessible pages may result in privacy violations, reputational damage, and potential regulatory non-compliance. Even inadvertent exposure increases organizational risk.

Remediation

  • Remove all sensitive data from public-facing responses.
  • Ensure backend logic does not expose PII to unauthenticated users.
  • Apply strict data-handling and output-filtering controls.

This report documents only the two specified findings based on observed scan evidence.
